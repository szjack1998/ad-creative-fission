# 爆款素材拆解裂变器 — 创意成因分析 × 脚本裂变工作站

## ⚡ 核心前提：全自动，零手动准备

**本 skill 不需要你准备任何 Excel / CSV / 后台导出文件。**

素材视频和消耗数据通过 `tencent-ads-assistant`（妙问）API **自动拉取**，文案通过多模态**自动拆解**，下游分析脚本直接吃自动产出的数据文件，一键出报告。整个链路对使用者是"拉数据 → 出报告"的闭环。

> 历史兼容：若你手头已有剪辑团队的 `脚本库.xlsx`，仍可走旧版 `fission_tool.py` 命名匹配裂变（见模块二末尾），但**新流程完全不依赖它**。

> 👉 **第一次用 / 小白？** 先看仓库里的 [`QUICKSTART.md`](QUICKSTART.md) —— 把"装什么、连什么、怎么跑"拆成 6 步清单，照着勾就行。

## 这个 skill 是什么

一站式广告创意优化框架，融合两块能力：
1. **成因分析**（源自 CreatiBI 方法论）：把创意变量化，用妙问拉取的真实投放数据指出"哪类变量有效/无效"。
2. **脚本裂变**（实战沉淀）：以妙问拉取的素材消耗数据为权威源、以多模态自动拆解的通顺文案为脚本源，定位 winning 母本，在「钩子/人群/利益/大字报」四维批量裂变新脚本，输出可直接交给剪辑团队的方案。

适用场景：教育、本地生活、电商、App 投放等任何"短视频素材 + 投放数据"的闭环。本 skill 只做分析/创作，不代投、不改平台数据。

## 模块一：创意成因分析框架

### 核心理念（必须内化）
- **创意变量化**：素材 = 一组变量取值的组合（情绪钩子/叙事节奏/视觉锚点/场景/角色）。
- **3 秒定胜负**：开头钩子决定点击意愿，情绪触发是核心。
- **创意即定向**：同一钩子在不同行业/渠道表现不同，要按渠道拆解。
- **成因分析**：用真实数据指出有效/无效变量组合，不靠猜。
- **方法论沉淀**：复盘结论变成团队资产。

### 七维标签体系（素材库打标签模板）
1. **产品信息**：品类 / 核心玩法 / 投放平台 / 目标人群 / 投放阶段 / 时间
2. **文件规格**：时长 / 格式 / 分辨率 / 版式 / 尺寸
3. **叙事结构**：框架(三段式/反转) / 节奏 / Hook / 核心信息 / 转场 / CTA
4. **内容玩法**：核心循环 / 机制清晰度 / USP / 难度感知 / 社交福利
5. **视觉表现**：美术风格 / 色彩 / 镜头 / 核心呈现物 / VFX / 对比手法
6. **听觉体验**：BGM / 音效 / 旁白风格·语速
7. **信息传达**：核心主旨 / 调性 / 激发情绪 / 行动动机 / 价值主张

### 变量维度（归因拆解表）
| 维度 | 含义 | 取值示例 |
|------|------|----------|
| 情绪钩子 | 用户为何停下 | 爽感 / 惊讶 / 紧张 / 好奇 / 恐惧 / 稀缺 |
| 叙事节奏 | 时间结构 | 3秒冲突开场 / 10秒反转 / 15秒收尾 |
| 视觉锚点 | 记忆点 | 数值跳动 / 红字提示 / 人脸 / 大字报 |
| 场景变量 | 环境 | 室内 vs 室外 / 现实 vs 二次元 |
| 角色属性 | 人物 | 性别 / 职业 / 年龄 / 真人 vs 数字人 |

### 5 步工作流（全自动版）
1. **采集（全自动）**：通过 `tencent-ads-assistant` 妙问 API 自动拉取素材ID/CTR/CVR/ROI/花费/转化，**无需导出或手工准备文件**。
2. **结构化打标**：对每条素材套七维标签 + 变量维度，产出结构化表（可由多模态读图自动初稿 + 人工校准）。
3. **变量聚合**：按单一变量聚合多条素材表现，算均值/中位数。
4. **成因分析**：找 Top/Bottom 变量组合，给"有效/无效"结论。
5. **输出 & 沉淀**：优化建议 + 修订标签库。

### 输出模板
```
## 创意成因分析报告
- 数据范围：渠道 / 时间 / 素材数（来自妙问API自动拉取）
- 总体表现：CTR x% / CVR y% / ROI z
- Top 变量组合（有效）：…
- Bottom 变量组合（无效）：…
- 优化建议（3-5 条，可执行）：…
- 标签库更新：新增 / 调整
```

## 模块二：脚本裂变工作流（全自动 · 妙问数据驱动）

### 数据采集（全自动 3 步，零手动准备）

| 步骤 | 动作 | 产出 | 谁来干 |
|------|------|------|--------|
| ① 拉数据 | 用 `tencent-ads-assistant` 妙问 API（DATA_QUERY）自然语言请求"拉取我的视频素材列表及消耗/CTR/CVR"，保存返回为 `material_video.json` | `material_video.json`（字段：视频链接、花费(元)、曝光数、点击数、CTR(%)、CVR(%)、转化量…） | **全自动（妙问API）** |
| ② 下载视频 | `python batch_extract.py` 从 material_video.json 提取 video_url 下载 | `frames_batch/<ID>/video.mp4`（断点续传） | **全自动（脚本）** |
| ③ 拆解文案 | 对下载视频多模态读图 + OCR，自动产出结构化文案与标签 | `videos_multimodal_fixed.csv`（通顺文案、钩子类型、视频类型、证书类别、真实消耗） | **全自动（多模态）** |

> ✅ 这 3 步完成后，下游归因与裂变所需的**全部输入已就绪**，无需任何人工填表。
> 📌 `videos_multimodal_fixed.csv` 就是旧版"脚本库 xlsx"的**自动化替代物**——文案不再靠人工整理，由多模态从真实视频里拆出来，且自带真实消耗字段。

### 变量归因（视频数据 × 脚本内容交叉）
典型可量化结论维度（基于自动拉取的消耗 + 自动拆解的文案）：
- **钩子类型**：好奇悬念 > 新奇吸引 > 权威背书（用 CTR×CVR 双指标判断）
- **人群定向**：通用+紧迫 > 特定人群锚定（高点击零转化往往说明人群锚定吸引非目标用户）
- **利益点密度**：3个硬利益（双休/五险/及格易）> 模糊利益（"提升自己"）
- **视觉形式**：真人/混剪视频 > 纯数字人（数字人"新奇感"常吸引非目标）
- **CTA 强度**：条件式引导（"看看你是否符合"）> 弱引导（"来直播间看看吧"）

### 四维裂变生成
以 **winning 母本**（效率最高素材的脚本结构，由自动消耗数据定位）为基，结构 = `好奇紧迫钩子 + 3个硬利益 + 条件式CTA`，在 4 维裂变：
1. **钩子角度**：行为中断 / 社交证明 / FOMO / 错失恐惧 / 反常识
2. **人群变体**：通用 / 宝妈 / 打工人 / 35岁+ / 转行族
3. **利益重组**：副业接单 / 事业单位福利 / 时间自由 / 行业缺口
4. **大字报短句**：紧迫窗口 / 利益清单 / 一句话击穿（适合图文/低成本版位）

### 优先级分级
- **P0 紧急生产**（4条）：复制 winning 结构换场景/开场，验证跨人群稳定性
- **P1 A/B 测试**（6条）：焦虑/社会认同/爽感钩子对比，含图文扩展（CPM 最低）
- **P2 边界探索**（5条）：数字人二次重试（加强利益密度）/ 权威背书 / 极简快剪
- **大字报短句**（3条）：纯文字卡片式，低成本版位专用

### 每条裂变脚本交付格式
```
[P0-1] 钩子标题
  正文：……
  变量标签：行为中断 / 时间锚定 / 3硬利益 / 条件CTA
  测试假设：换场景验证 yg11 结构在 X 人群的 CVR 稳定性
```

### 输出物（HTML 报告）
包含：① 视频-脚本匹配表 ② 变量归因表 + 图表 ③ 裂变脚本矩阵（P0/P1/P2/短句分Tab）④ 30天执行排期（生产→测试→放量→沉淀）⑤ 合规红线提醒。直接 `Write` 成 HTML 并 `present_files` 打开预览。

### 历史兼容（可选）：自带 Excel 脚本库时
若确有现成 `脚本库.xlsx`（一列"文案"，含原始/裂变脚本/大字报短句），可用旧版 `fission_tool.py` 走命名匹配裂变：
```bash
python fission_tool.py --scripts 脚本库.xlsx --videos material_video.json --output html
```
命名匹配规则（仅旧版生效）：`yg/yg-*`→好奇悬念钩子；`yyy/yyy-*`→权威背书；`数字人`→AI口播；`图文`→大字报短句。新流程推荐直接用自动拆解的 `videos_multimodal_fixed.csv`，不再依赖此匹配。

## 模块三：动态视频深度分析（Dynamic Video Analysis）

### 为什么需要这个模块

传统素材分析只看「文案说了什么」（OCR/多模态读图），丢失了**时间维度**和**视觉叙事**。动态分析回答的是：
> 这段视频是怎么拍的？为什么这样拍有效？哪个设计决策在驱动表现？

对标专业工具（如 CreatiBI/SIG 框架）的拆解粒度：时序分段 → 画面语义 → 叙事结构 → 心理机制 → 可执行建议。

### 核心能力（三层数据源，全部自动）

| 层 | 数据来源 | 能力 | 输出 |
|---|---|---|---|
| **L1 程序化特征** | OpenCV 直接分析 mp4（视频来自 [2] 自动下载） | 转场检测(帧间直方图差分)、运动幅度(帧差)、色彩风格、节奏曲线 | dynamic_analysis.json |
| **L2 时序视觉** | 按时间段密集抽帧 → 多模态逐段读图 | 每段画面内容、转场方式、叙事作用、情绪曲线 | VISUAL_CLASSIFICATION 字典 |
| **L3 投放数据映射** | 妙问API material_video 真实指标（[1] 自动拉取） | CTR/CVR/花费/点击/转化 × 视觉格式交叉归因 | 综合报告 |

### 工作流（5 步闭环，数据全自动）

#### Step 1: 批量下载视频（来自妙问自动拉取）
`material_video.json` 由妙问API自动拉取后，批量下载到本地：
```bash
# 从 material_video.json 提取URL并下载到 frames_batch/<ID>/video.mp4
python batch_extract.py   # 已有脚本，支持断点续传
```

#### Step 2: 时序分段抽帧（dynamic_frames.py）
把每个视频按时轴切成 N 段，每段密集采样拼成横向 grid（保留时序信息）：

```bash
# 单个视频
python dynamic_frames.py <素材ID> [分段数=4] [每段帧数=6]

# 批量全部视频
python batch_segments.py    # 自动遍历 frames_batch/*/video.mp4
```

输出：`frames_batch/<ID>/segments/seg1_0-6s.png`, `seg2_6-11s.png`, ...

**关键改进 vs 旧方案**：
- ❌ 旧：N 帧压成一张大 grid（丢失时间顺序）
- ✅ 新：按时序分成独立小图（保留开场→铺垫→收尾的叙事链）

#### Step 3: 程序化动态分析（dynamic_analyzer.py）

OpenCV 对每个 mp4 做底层特征提取：

```bash
python dynamic_analyzer.py
# → dynamic_analysis.json (完整特征)
# → dynamic_analysis.csv
```

每条输出的核心字段：
```json
{
  "material_id": "EDU-V01",
  "duration_s": 19.0,
  "fps": 30.0,
  "resolution": "720x1280",

  // === 转场 ===
  "transitions": [],           // 真实切镜时刻点列表(秒)
  "transition_count": 1,        // 切镜次数
  "is_single_shot": true,       // 是否单机位无切换

  // === 节奏 ===
  "avg_motion": 14.3,           // 平均运动幅度(像素差)
  "peak_motion": 45.2,          // 峰值运动
  "motion_stability": 0.82,     // 运动稳定性(低=稳定/高=波动)
  "rhythm_curve": {0: 12.3, 1: 8.7, ...}, // 每秒运动强度

  // === 色彩 ===
  "avg_brightness": 0.62,       // 平均亮度(0~1)
  "avg_saturation": 0.38,       // 平均饱和度
  "dominant_rgb": [180,170,160],// 主色调RGB

  // === 风格自动标签 ===
  "style_tags": ["单机位固定镜头"],
  "style_summary": "单机位固定镜头"
}
```

**风格自动分类规则**（opencv 纯数据驱动）：
| 条件 | 标签 |
|---|---|
| 亮度>0.65 且 饱和度<0.35 | 亮色/白底 |
| 饱和度>0.45 | 鲜艳/彩色 |
| 平均运动>15 | 高动态/快剪 |
| 转场数≥2 | 多场景切换 |
| 以上都不满足且单镜头 | 单机位固定镜头 |

#### Step 4: 多模态逐段视觉验证（人工/AI 读图）

对程序化分析结果做定性补充——这是 L1 数据做不到的：
- **视觉格式分类**（户外真人/纯图文/多场景混剪/科技风开场…）
- **时序叙事拆解**（每段画面内容+叙事作用）
- **核心视觉锚点识别**（证书/手机/大字报颜色策略）
- **特殊发现标注**（如"仅7秒超短但CVR最高"）

操作方式：用 Read 工具逐段读 `segments/seg*.png`，输出结构化描述写入 VISUAL_CLASSIFICATION 字典。

#### Step 5: 生成交互式报告（render_dynamic_report.py）

合并 L1+L2+L3 数据生成纯静态 HTML：

```bash
python render_dynamic_report.py
# → xincheng_dynamic_analysis_report.html (base64内嵌图片，零JS)
```

报告包含：
- 总览统计卡片（素材数/总花费/品类分布/格式类型数）
- **视觉格式分布表**（每种格式多少条、对应什么表现）
- 每条素材详情卡片：
  - 基础指标（花费/CTR/CVR/点击/转化）
  - 动态特征（时长/转场次数/运动幅度/亮度/拍摄方式标签）
  - 视觉分类（格式/描述/色彩基调/核心视觉锚点）
  - 时序拆解（每段画面的叙事作用列表）
  - 特殊发现标注（红色高亮）
  - 缩略图预览

### 实战发现的视觉格式分类（示例客户 · 19 条素材，数据已脱敏）

| 格式 | 数量 | 代表素材 | 典型特征 | 表现规律 |
|------|------|----------|----------|----------|
| **户外真人+证书展示** | 2 | EDU-V01, EDU-V02 | 自然光+手持证书+手势丰富 | 花费高+CVR较高(约3~5%)——信任感最强 |
| **白底口播+动态字卡** | 10+ | EDU-V05, EDU-V07 等 | 白底单机位+全屏字幕/大字报切换 | 主力形式，CTR高但CVR分化 |
| **纯图文信息卡** | 2 | EDU-V03, EDU-V04 | 绿/米黄底+文字卡片轮播 | CVR较高(约5%)！超短(7~19s)直投信息 |
| **多场景混剪** | 2 | EDU-V17, EDU-V18 | ≥3个独立场景硬切 | 运动幅度高，CVR也不错(约2~5%) |
| **科技风开场→口播** | 1 | EDU-V06 | 暗场科技蓝(~2s)→切白底 | CTR高(约6%)但CVR低(约0.4%)——钩子太泛 |

### 关键洞察（来自动态分析，示例数据已脱敏）

1. **户外真人 > 白底口播 > 科技风开场**（CVR 排序）
2. **纯图文 = 最低成本 + 最高 CVR 效率**（超短直投信息不废话）
3. **场景硬切促进转化**（多场景混剪类 CVR 普遍高于单镜头口播）
4. **CTR 高 ≠ 好**（科技风开场 CTR高但CVR低，吸引非目标人群）

## 配套脚本工具一览

| 脚本 | 功能 | 用法 |
|------|------|------|
| `batch_extract.py` | 从妙问拉取的 material_video.json 提取URL并下载视频（断点续传） | `python batch_extract.py` |
| `dynamic_frames.py` | 单个视频按时序分段抽帧 | `python dynamic_frames.py <ID> [分段数] [每段帧数]` |
| `batch_segments.py` | 批量给所有本地视频生成分段帧 | `python batch_segments.py` |
| `dynamic_analyzer.py` | opencv程序化动态分析全部视频 | `python dynamic_analyzer.py` → dynamic_analysis.json |
| `render_dynamic_report.py` | 合并L1+L2+L3数据生成HTML报告（动态分析） | `python render_dynamic_report.py` |
| `real_fission.py` | 真实数据版成因分析+裂变(妙问数据驱动，不需Excel) | `python real_fission.py` |
| `real_fission_v2.py` | **制片级裂变**：绑定已验证高效视觉格式+镜头脚本+口播文案+测试假设 | `python real_fission_v2.py` |
| `fission_tool.py` | 【历史兼容】Excel脚本库 × 视频数据 → 成因分析+裂变 | `python fission_tool.py --scripts xlsx --videos json --output html` |
| `rebuild_v3_static.py` | 纯静态HTML渲染器(通用模板) | `python rebuild_v3_static.py` |

**依赖**：
- `pip install opencv-python-headless numpy`（dynamic_frames/dynamic_analyzer/动态分析）
- 文案拆解与数据拉取依赖妙问API + 多模态读图，**无需手动准备文件**
- 无需外部模型 API（opencv 本地计算）

## 与现有工具协同
- **数据采集**：`tencent-ads-assistant`（妙问）自动拉取素材列表与消耗 → `material_video.json`，再自动下载视频、多模态拆解文案 → `videos_multimodal_fixed.csv`。**全程无需人工导出/填表**。
- 素材批量生产 → 用户 pyJianYingDraft / 剪映，本 skill 负责"先定义变量、后量化复盘"。
- 拒审复盘 → 记录拒审原因到标签库，下次规避同类元素。
- 多模态读图 → WorkBuddy 内置 Read 工具直接读 PNG/JPG（支持视觉理解）。

### 模块间数据协同链路（重要，避免两步脱节）
本 skill 三个分析模块是**串联流水线**，上游产物是下游输入，文案数据必须跨模块继承：

```
[妙问API自动拉取]         [多模态自动拆解]           [OpenCV自动分析]           [报告生成]
material_video.json  ──▶  videos_multimodal     ──▶  dynamic_analysis     ──▶  HTML 报告
(素材+消耗,自动)           _fixed.csv(文案,自动)       .json(特征,自动)          (可交付)
```

- **协同红线**：动态分析报告的卡片模板**必须包含"通顺文案"块**（来自 `videos_multimodal_fixed.csv` 的 `通顺文案` 列）。动态分析是"在文案基础上加视觉层"，不是另起炉灶把文案丢掉。
- **数据同源（全自动）**：真实消耗（CTR/CVR/花费）统一以妙问API拉取的 `material_video.json` 为权威源；文案以 `videos_multimodal_fixed.csv` 为权威源（多模态自动拆解，非人工整理）。
- **交付物关联**：`xincheng_dynamic_analysis_report.html`（动态+文案）、`xincheng_real_fission_report.html` / `xincheng_real_fission_v2_report.html`（裂变）三者共享同一批素材ID，可交叉对照。

## 注意事项
- 只做分析与建议，不代投、不改平台数据。
- 变量结论需足够样本量；小样本明确标注"样本不足"。
- 区分"相关"与"因果"，避免过度推断。
- 标签体系先小范围跑通再扩充，避免"标签地狱"。
- **本 skill 默认全自动**：素材与消耗来自妙问API、文案来自多模态自动拆解，使用者无需准备任何 Excel/CSV。`fission_tool.py` 的 xlsx 路径仅为历史兼容，非必需。
- **HTML 报告必须用纯静态渲染**（零 JS），避免 file:// 安全限制导致空白问题。
- 图片嵌入优先用 base64 内嵌（自包含），或配合 localhost HTTP 服务器访问相对路径图片。
